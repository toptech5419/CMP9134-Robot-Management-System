HCI Laws & Principles Applied — Summary
Fitts's Law MOVE + E-STOP are the two physically largest interactive elements on the entire dashboard
Hick's Law Move panel has exactly 2 inputs — no dropdown menus, no extra choices
Shneiderman Rule 1 Same layout, fonts, icons used across all screens
Shneiderman Rule 3 Every action (click, success, error, retry) produces instant visual feedback
Shneiderman Rule 4 Success notification + log entry together confirm full task closure
Shneiderman Rule 5 Client-side validation prevents out-of-range commands reaching the API
Shneiderman Rule 6 Reset button allows reversal — inputs can be cleared
Shneiderman Rule 7 Role badge always visible — user always knows their permission level
Shneiderman Rule 8 Log paginated — dashboard minimal, no cognitive overload
Norman: Visibility Viewer gets "Read-Only" block — no mystery buttons
Norman: Error Design Every error state shows recovery path, not just a red box
Norman: Mapping Battery bar fills left-to-right — matches mental model of "filling up"
